# e-Tax ハンドオフ運用 Runbook — 生成 .xtx を e-Taxソフト(WEB版)で取込・署名・送信 (#80)

本プロジェクトは青色申告決算書(一般用)を**実 e-Tax 交換ファイル `.xtx`(官式 KOA210)** として
生成するところまでを担う(#79 で .xsd 形式妥当性を機械保証済み)。**税額計算・電子署名・送信は
作らない。** 生成した `.xtx` を、ユーザーが**国税庁 e-Taxソフト(WEB版)に取り込み、マイナンバー
カードで署名して送信する** —— この 80/20 ハンドオフ経路が公式に成立することはリサーチで裏取り
済みであり、本書はその手順・チェックリスト・スコープ境界を定める。

> この経路の最後の「実際に取り込める」確認だけは、**人間 + マイナンバーカード + e-Taxソフト**が
> 必須で自動化できない(本リポで唯一)。実取込の受入結果は本書の
> [受入確認の記録](#受入確認の記録-manual-acceptance) に記録する。

## スコープ境界(本プロジェクトの対象 / 対象外)

| 領域                                        | 担い手                                 | 対象         |
| ------------------------------------------- | -------------------------------------- | ------------ |
| 仕訳・元帳・青色申告決算書(数値)の作成      | **ai-books**                           | ✅           |
| 決算書 → `.xtx`(KOA210)生成 + .xsd 形式検証 | **ai-books**(`export_etax`)            | ✅           |
| `.xtx` の取込・読込確認(e-Taxソフト WEB版)  | ユーザー(手動・本書の手順)             | ✅(手順提供) |
| **確定申告書B 本体**(所得税の計算・税額)    | 公式ツール / 下流ツール                | ❌           |
| **税額計算**                                | 公式ツール / 下流ツール                | ❌           |
| **電子署名の付与**                          | e-Taxソフト(WEB版)+ マイナンバーカード | ❌           |
| **受付システムへの送信**                    | e-Taxソフト(WEB版)                     | ❌           |
| **利用者識別番号 / 電子証明書の取得**       | ユーザー(e-Tax 利用開始手続)           | ❌           |

> ⚠️ **対象外の境界**: 本プロジェクトは「ledger + 青色申告決算書 + e-Tax 取込データ」を生成する。
> **税額計算・確定申告書B本体・電子署名・送信・利用者識別番号/電子証明書取得は本プロジェクトの
> 対象外**(公式ツール/ユーザー側)であり、コードにも本書にも実装・代行は含めない
> (参照: [README#out-of-scope](../../README.md))。

## なぜ e-Taxソフト(WEB版)「作成済みデータの利用」ルートなのか

第三者(本プロジェクト)が生成した青色申告決算書データの取込先は **e-Taxソフト(WEB版)の
「作成済みデータの利用」**を採る。**確定申告書等作成コーナーの一括添付ルートは採れない**。

- **制約(一次情報)**: 確定申告書等作成コーナーで確定申告書と「一緒に送信できる」青色申告決算書
  データは「**決算書・収支内訳書作成コーナーで作成した電子申告等データに限る**」と国税庁が明記して
  いる(<https://www.keisan.nta.go.jp/r1yokuaru/cat1/cat15/cid179.html>)。本プロジェクトの `.xtx` は
  作成コーナー由来ではないため、**作成コーナーへの一括添付ではなく e-Taxソフト(WEB版)取込ルート**
  を採用する。
- **このルートで署名・送信は公式ツール側で完結**するため、本プロジェクトに署名・送信の自前実装は
  不要(e-Tax の Web-API は参照系のみ <https://www.e-tax.nta.go.jp/shiyo/api.htm>)。

## 事前準備(ユーザー側)

- [ ] **マイナンバーカード**(電子証明書 有効)。署名・本人確認に必須。
- [ ] **IC カードリーダ** または **マイナポータルアプリ対応スマートフォン**(QR/かざし読取)。
- [ ] **利用者識別番号**(未取得なら e-Tax の利用開始届出で取得。本プロジェクト対象外)。
- [ ] 推奨環境(対応ブラウザ + 事前準備セットアップ)の導入。e-Taxソフト(WEB版)の案内に従う。
- [ ] ai-books で生成した **`.xtx` ファイル**(次節)。

## 手順 1: `.xtx` を生成する(ai-books)

`export_etax` MCP tool を `format="xtx"` で呼ぶと、対象年度の青色申告決算書が官式 KOA210 の
`.xtx`(実 e-Tax 交換ファイル)として返る。

```text
export_etax(fiscal_year=2025, format="xtx")
```

- `format="xtx"` が**実申告に渡す形式**。`csv` / `xml` は人間確認・デバッグ用の補助様式で、
  e-Tax には取り込めない。
- 出力前に必須項目・桁数・勘定科目コード・月をスキーマ検証し、欠落/不正は `EtaxValidationError`
  で**全件まとめて**エラーになる(金額は整数円、端数は不可)。エラーが出たら帳簿側を直してから
  再生成する。
- 形式妥当性は #79 の .xsd 検証で機械保証済み(`tests/test_etax_xtx.py` / CI `etax-xsd`)。
- 返ってきた XML を**拡張子 `.xtx`** でローカルに保存する。

> 🔒 **生成物はコミットしない。** `.xtx` は事業者の確定数値(秘密情報)を含む。リポジトリには
> 絶対に置かず、`.ai-books/` 配下や gitignore 済みのパスに保存すること
> (参照: [README#e-tax-取込データ出力](../../README.md))。

## 手順 2: e-Taxソフト(WEB版)で取り込む → 署名 → 送信

公式手順(FAQ <https://www.e-tax.nta.go.jp/toiawase/qa/e-taxweb/49.htm> /
<https://www.e-tax.nta.go.jp/toiawase/qa/e-taxweb/31.htm>)。**各ステップはスクリーンショットを
残す**ことを前提にチェックリスト化する。

- [ ] **1. ログイン**: e-Taxソフト(WEB版)にマイナンバーカード方式または利用者識別番号でログイン。
- [ ] **2. 「申請・納付手続を行う」** を選ぶ。
- [ ] **3. 「作成済みデータの利用」** を選ぶ。
- [ ] **4. 「操作に進む」** を押す。
- [ ] **5. ファイルを参照** し、手順 1 で保存した **`.xtx` を選んで読込**。
- [ ] **6. 読込成功を確認**(エラーなく帳票が表示/受理される)。← **本 Issue の受入確認点**。
      ここで版不一致や必須欄不足が出たら [トラブルシュート](#トラブルシュート) 参照。
- [ ] **7. マイナンバーカード認証**(本人確認)。
- [ ] **8. 電子署名の付与**(公式ツール側)。
- [ ] **9. 受付システムへ送信**(公式ツール側)。← **テスト運用では送信しない**(下記)。
- [ ] **10. 送信後**: 受付結果(受信通知)を確認・保存。

> テスト運用(リリース前の受入確認)では **手順 6 の「読込成功」までで止め、送信(手順 9)は
> 行わない**。手順 7〜10 は本番申告時の流れとして記す。

## 既知の制約・年度差異

- **帳票版は年度ではなく様式改定で増える**。令和7年分の現行版は **KOA210 v11.0(一般用)**
  (様式文書日付 2023-09-27 だが令和7年10月30日確定版パッケージ内の最高版 = 現行)。不動産用
  KOA220 / 農業用 KOA240 は v8 で安定(参照: [docs/etax/README.md](./README.md))。
- 本プロジェクトの `.xtx` は **一般用 KOA210 のみ**を生成する(製造原価は KOA210 に内包)。不動産
  所得用・農業所得用には未対応。
- **65万円控除**は e-Tax による電子申告 + 優良な電子帳簿等の要件を満たす必要がある
  (参照: [docs/etax/aoiro-65man-requirements.md](./aoiro-65man-requirements.md))。要件充足は
  ユーザー側の運用責任。
- e-Taxソフト(WEB版)の画面遷移・名称は国税庁の更新で変わりうる。手順がずれたら公式 FAQ
  (上記リンク)を一次情報として読み替える。

## トラブルシュート

| 症状                         | 想定原因                                                  | 対処                                                                                                                                        |
| ---------------------------- | --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| 読込時に**版不一致**エラー   | e-Tax 側の受付版と `.xtx` の様式版(`VR`)がずれ            | 対象年度の現行版を確認(README の帳票版表)。様式改定があれば該当様式の `*_layout.json` を再生成(#79/#103 の再現手順)してから再生成・再取込。 |
| 読込時に**必須欄不足**エラー | 帳簿側にヘッダ必須メタ(住所/氏名/提出年月日/業種等)が無い | 該当項目を帳簿側で補い、`export_etax` を再実行。`EtaxValidationError` が出る場合は全件メッセージに従って修正。                              |
| ファイルを認識しない         | 拡張子が `.xtx` でない / 文字コード                       | 保存時に拡張子 `.xtx`、UTF-8 を確認。`export_etax(format="xtx")` の出力をそのまま保存する。                                                 |
| 署名・送信に進めない         | マイナンバーカード/カードリーダ/利用者識別番号            | 事前準備の各項目を確認(本プロジェクト対象外。e-Tax の案内に従う)。                                                                          |
